# fantasy-football-analytics — Codebase Guide

## Overview

This is a **machine learning + data engineering project** for predicting fantasy football RB PPG using XGBoost, Optuna tuning, and SHAP explainability. The pipeline ingests 10 years of nfl_data_py seasonal data, enriches it with 61 engineered features (snap counts, depth charts, injury history, NGS rushing metrics), and trains a walk-forward validated model.

**Core files:**
- `RB_Dataset_Builder.ipynb` — 57-cell notebook: data loading, cleaning, feature engineering
- `Fantasy Football 2026 Claude.ipynb` — EDA, exploratory analysis, trending demo
- `rb_features_2015_2025.csv` — final dataset: 1,229 rows (RB-seasons), 71 columns
- `rb_features_2015_2025.parquet` — same data, parquet format (fastparquet engine)

**Output data (generated by model training):**
- `model_output/rb_ppg_xgb.json` — trained XGBoost model (Booster)
- `model_output/rb_predictions_2025.csv` — 2025 predictions: player_name, ppg_predicted_2025, rank_2025, tier
- PNG visualizations: validation_summary, shap_beeswarm, shap_bar, pred_vs_actual_YYYY

## Key Technical Context

### Data Pipeline (`RB_Dataset_Builder.ipynb`)

**Data sources:**
1. `nfl_data_py.import_seasonal_data()` — base seasonal stats (2015–2024, 2025 not yet available)
2. `import_pbp_data()` — chunked at 4-year windows to limit memory; extracts goal-line touches (yardline ≤ 20) and team rushing context
3. `import_snap_counts()` — snap percentages via PFR pfr_player_id → gsis_id crosswalk
4. `import_depth_charts()` — week 1 RB depth positions
5. `import_injuries()` — out/limited games per season
6. `import_ngs_data(stat_type='rushing')` — RB-specific NGS rushing metrics (receiving NGS has no RB data)
7. `import_ids()` — gsis_id ↔ sleeper_id crosswalk (77% coverage, used only for snap_counts join)

**Critical discovery:** `import_seasonal_rosters()` already contains native sleeper_id + birth_date columns → 99.4% match rate without explicit ID join.

**Feature engineering:**
- Per-game conversion: divide season totals by games_played
- aDOT: air_yards / targets (average depth of target)
- Carry/target share: player's carries / team's carries
- YoY deltas: season Y value − season Y−1 value (for returning players)
- Goal-line PPG: goal_line_carries_pg, goal_line_targets_pg, goal_line_touches_pg
- NGS metrics: ngs_efficiency, ngs_ryoe_per_att, ngs_rush_pct_over_expected, etc.

**Missing features (PBP skipped by default):**
- carries_share, gl_carries_pg, gl_targets_pg, gl_touches_pg (100% NaN if SKIP_PBP=True)
- Affects model ceiling but acceptable for baseline; full PBP run (~40 min) populates these

### Model Training

**Feature set (62 features in MODEL_FEATURES list):**
- Prior-season PPG (strongest single predictor)
- Per-game stats: carries, targets, yards, TDs, EPA, first downs, fumbles (rushing + receiving)
- Efficiency: ypc, ypr, catch_rate, scrimmage_yds_per_touch, aDOT
- Situational: snap %, depth chart position, goal-line touches, team rushing context
- NGS: efficiency, % ≥8 defenders, avg time to LoS, RYOE/attempt, rush % over expected
- Player attributes: age, years_exp, height, weight, draft_number
- Injury: games out/limited (current + prior season)
- Deltas: YoY changes in PPG, carries, targets, yards, EPA, snap %

**Model architecture:**
- XGBoost with Optuna hyperparameter tuning (60 trials)
- Objective: minimize negative Spearman ρ (correlation ranking metric)
- Training data: 2015–2020 (tuning) or season < test_year−1 (walk-forward)
- Validation: 2021 (tuning) or season == test_year−1 (walk-forward)
- Test folds: 2019, 2020, 2021, 2022, 2023, 2024 (6-fold walk-forward)

**Evaluation metrics:**
- Spearman ρ (primary) — ranking correlation
- MAE, RMSE — absolute/squared error
- Top-K precision @ K=12, K=24 — % of top K predictions in top K actuals
- Baseline: prior-season PPG

**Results:**
- Avg Spearman ρ: 0.732 (baseline: 0.721)
- Avg MAE: 2.91 PPG (vs 3.15 baseline)
- Top-24 precision: 72.9%
- 2025 predictions: 113 returning RBs ranked by ppg_predicted_2025 with RB1/RB2/RB3/Depth tiers

### Known Issues & Workarounds

1. **numpy/pandas versioning:** nfl_data_py requires numpy<2, pandas<2; shap requires numpy≥2. **Workaround:** venv has numpy 2.4.4 + pandas 3.0.2. nfl_data_py still imports but with version warnings. Avoid re-running nfl_data_py if possible; reuse cached CSV/parquet.

2. **2025 seasonal data unavailable:** nfl_data_py doesn't have 2025 data yet (as of May 2026). Notebook gracefully skips with warning.

3. **NGS receiving has no RB data:** `import_ngs_data(stat_type='receiving')` returns only WR/TE. Use `stat_type='rushing'` for RB metrics instead.

4. **PBP chunking required:** Full PBP load (2015–2025, ~17 weeks × 10 years) exceeds memory. Solution: load 4-year chunks, extract goal-line + team carries, delete chunk.

## How to Extend

### Adding a new position (WR/QB)

1. **Update feature config:** Filter `import_seasonal_data()` by position='WR' in the loader
2. **Adjust features:** Some stats are position-specific (WR: target share more relevant; QB: completion %, EPA more critical)
3. **Retrain:** Walk-forward validation on position-specific training set
4. **Update SHAP:** SHAP charts will reflect position-specific feature importance

### Rookie model

- Separate training set: college stats (ESPN, NFL Draft Scout) + draft metrics (round, ADP, height, weight, 40-time, cone drill, vertical)
- Target: rookie-year PPG (season 1 only)
- Model: XGBoost with similar walk-forward structure
- Challenge: smaller dataset (~250 rookies/year), noisier target

### REST API

- Framework: FastAPI or Flask
- Endpoint: `POST /predict` with player stats, returns ppg_predicted, rank, tier
- Model loading: `xgb.Booster().load_model('rb_ppg_xgb.json')`
- Deployment: Docker + Cloud Run / Heroku

## Testing & Validation

**Walk-forward folds (6 years):**
```
Fold 1: train 2015–2017, test 2018 → eval on 2018 actuals
Fold 2: train 2015–2018, test 2019 → eval on 2019 actuals
...
Fold 6: train 2015–2023, test 2024 → eval on 2024 actuals
```

**Baseline comparison:** last-year's PPG as naive predictor. XGBoost beats baseline on MAE, ties on Spearman ρ (model is modest improvement but valid).

**Feature importance:** SHAP beeswarm chart ranks features by absolute impact on predictions. Prior-season PPG dominates (as expected), followed by per-game stats and snap counts.

## Quick Reference: Column Definitions

See the 61-feature breakdown in README.md. Key derived columns:

- `ppg` — points per game (fantasy_points_ppr / games)
- `aDOT` — air yards / targets
- `carries_share` — player carries / team carries
- `is_rb1_wk1` — depth_team_wk1 == 1 (binary)
- `games_out_seas` — games missed due to Out/Doubtful
- `games_limited_seas` — games limited (Questionable/Limited)
- `ppg_yoy` — current season PPG − prior season PPG
- `ngs_ryoe_per_att` — rush yards over expected per attempt (pure skill metric)
